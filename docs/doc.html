How to use this website

  1. Header
    • Header will display this site name and login field.
    • If you haven't registered yet, you can create a new account.
      After click "New Account" button, you can go to add user page and create new account.
    • If you are a registered user, you can login with your username and password.
      After you are logged in, you can see your name, your job position(user, manager or admin), "Logout" and "Change PW" buttons.
      "Change PW" button allows you to change your password.
    • If you are an administrator for this site, you can see "User list" and "Add Manager" buttons as well.
      "Add Manager" button allows the administrator to add new manager.
      "User list" button allows the administrator to see all users.

  2. Employers home page
    • This page displays all employers.
    • When you click an employer name, then you will go to the employer detail page.

  3. Employer detail page
    • This page displays emplyer details such as industry, description and jobs offered by this employer.
    • When you click an job title, then you will go to the job detail page.
    • If you are a manager who work for this employer, you can see the text fields for offering new job. However, you are not a manager who work for this employer, you cannot see these text fields.
    • Only jobs offered by this employer will be displayed. If these jobs are more than 5, page will be linked.

  4. Job detail page
    • This page displays job details such as location, description and salary.
    • If you are a user, you can see "Apply" button.
      If you click this button, you will go to apply-job page.
    • If you are a manager who is offering this job, you can see applicants who applied for this job. Also, you can see the application if you click an applicant name.

  5. Application page
    • Application page displays job title, applicant name, application letter and applied date. Applicant name will be linked to applicant detail page.

  6. Users home page
    • This page displays a text field to search for a job and recently-added jobs.
    • You can search jobs or you can see all jobs as in assignment1.
    • Recently-added jobs will display only five jobs that have added recently.
    • If you click "View all jobs" button, you will go to job list page.

  7. Job list page
    • Job list page displays all jobs but it will display five jobs per page.

  8. Change password page
    • Change password page displays all details for you.
      If you have uploaded a picture, you will be able to see your picture.
    • In order to change your password, you have to enter new password in both Password and Re-type text field. If your password that you entered in each text field are not same, an alert window will show up.

  9. User list page
    • Administrator only can see this page.
    • User list page displays all users and each user name will be linked to user detail page.
    • If you click user name, you will go to user detail page.

  10. User detail page
    • User detail page displays all user details.
    • If you are an administrator, you can see "Change Position" button.
      This button allows the administrator to change job position from 'user' to 'manager'. (Also, job position can be changed from 'manager' to 'user'.
      When you click "Change Position" button, option box for changing position will show up.

  11. Apply job page
    • If you are a normal user and you clicked "Apply" button, you will go to apply job page.
    • If you already applied for this job, you can see this sentence "You already applied for this job!!".
    • Your name, job title and a text field for application letter will be displayed on this page.
    • If you click after you enter your application letter, you will go to application page.

  12. Footer
    • Footer displays Web Developer name and Date.

Schema for employers table


create table employer (
id int not null auto_increment,
  name varchar(80) not null,
  industry varchar(80),
  description text not null,
  PRIMARY KEY (id)
) engine=innodb;

Schema for jobs table


create table job (
  id int not null auto_increment,
  employer int references employer(id),
  title varchar(80) not null,
  description text not null,
  location varchar(80) not null,
  salary int not null,
  epoch timestamp not null,
  duedate timestamp not null,
  PRIMARY KEY (id)
) engine=innodb;

Schema for employees(users) table


create table employee (
  id int not null auto_increment,
  employer int references employer(id),
  username varchar(30) not null,
  password varchar(30) not null,
  email varchar(80) not null,
  firstname varchar(30) not null,
  lastname varchar(30) not null,
  category varchar(30),
  phoneno int not null,
  imagedata blob default "",
  imagename varchar(40) default "",
  imagetype varchar(40) default "",
  imagesize varchar(40) default "",
  PRIMARY KEY (username),
  UNIQUE KEY (id),
  KEY (password)
) engine=innodb;

Schema for applications table


create table application (
  id int not null auto_increment,
  applicant int references employee(id),
  firstname varchar(30) not null,
  lastname varchar(30) not null,
  letter text not null,
  jobID int references job(id),
  epoch timestamp not null,
  PRIMARY KEY (id)
) engine=innodb;








Usability Testing


In order for usability test, all functions as I did for assignment1 were tested. These all worked properly.
All text fields were tested and worked properly.
When I checked password update function, there was a problem that returned wrong information.
In order to correct this, I had to think more about conditional statement.
For inserting date function, I used timestamp but it didn't work.
I spend a lot of time and finally, I found the cause. It was because compatibility between date and timestamp.
Add new job and new user also worked as other functions.
While I was testing delete and update function, I found one problem that application caused some error when a job that related to the application deleted.
So, I created delete function for application. If a job is deleted, related application will be deleted. Although I have tried to find error, I haven't found any problem yet.
Nevertheless, I worry about possiblity that there is another problem that I haven't found.
